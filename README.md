# Healthper Portfolio

> 헬스장 운동기구에 IoT 기술을 적용하여 사용자에게 운동 데이터 측정, 자동 무게 조절, 1RM 기반 점수 계산을 지원하여
   사용자에게 안전하게 운동을 할 수 있게 도움을 주고 경쟁심리를 유도하여 사용자의 운동 흥미를 높임


### 데이터 출처

Healthper에서 사용하는 데이터는 운동기구에 부착된 IoT 센서와 사용자의 운동 입력값에서 발생합니다.

### 데이터 수집 흐름

사용자가 운동을 시작하면 하나의 운동 세션이 생성되고, 센서 데이터는 해당 세션에 연결되도록 설계했습니다.

flowchart LR
    A[QR인식/사용자 운동 시작] --> B[sessionId 생성]
    B --> C[운동기구 equipmentId 연결]
    C --> D[센서 sensorId 연결]
    D --> E[ADXL345 센서 데이터 수집]
    E --> F[운동 수행 데이터 정리]
    F --> G[1RM 기반 점수 계산]
    G --> H[사용자별 운동 기록 저장]


데이터 처리 기준

센서 데이터가 여러 기구에서 동시에 발생할 수 있기 때문에, 각 데이터에 식별값을 함께 저장하는 구조로 설계했습니다.

항목	역할
userId	어떤 사용자의 운동인지 구분
equipmentId	어떤 운동기구에서 발생한 데이터인지 구분
sensorId	어떤 센서에서 발생한 데이터인지 구분
sessionId	한 번의 운동 기록 단위를 구분
timestamp	데이터가 발생한 시간 기록

이 구조를 통해 센서 데이터가 섞이더라도 특정 사용자, 특정 운동기구, 특정 운동 세션에 해당하는 데이터만 점수 계산에 사용할 수 있도록 했습니다.

데이터 정제 및 점수화

수집된 센서 데이터와 운동 수행 데이터는 점수 계산에 필요한 값으로 정리됩니다.

처리 항목	설명
반복 횟수	운동 수행 중 감지된 움직임을 기준으로 계산
운동 무게	사용자가 선택한 무게 값 사용
자세 완성도	센서 기반 움직임 안정성을 기준으로 반영
예상 1RM	Epley 공식을 사용해 무게와 반복 횟수를 하나의 기준으로 환산
최종 점수	1RM 기반 기본 점수에 자세 완성도 보정값을 적용
예상 1RM = 운동 무게 × (1 + 반복 횟수 ÷ 30)
최종 점수 = 기본 점수 × (0.9 + 자세완성도 / 100 × 0.1)
Data 관점에서의 프로젝트 의의

Healthper는 단순히 운동 결과를 저장하는 것이 아니라, 센서 데이터와 사용자 운동 데이터를 세션 단위로 구분하고 점수화하는 구조를 설계한 프로젝트입니다.

특히 userId, equipmentId, sensorId, sessionId를 기준으로 데이터를 분리하여 여러 센서 데이터가 동시에 발생할 수 있는 환경에서도 사용자별 운동 기록을 관리할 수 있도록 설계했습니다.

## Features
- 자동 무게 조절: 운동 결과에 따라 다음 운동 무게를 증가, 유지, 감소하도록 설계
- 점수 측정 알고리즘: Epley 1RM 공식을 기반으로 무게와 반복 횟수를 점수화
- 센서 기반 측정: ADXL345 가속도 센서를 활용해 운동 수행 데이터를 측정

## Architecture
Healthper는 헬스장 내 모든 운동기구를 자동화하는 시스템이 아니라,
반복 동작이 명확한 일부 기구에 IoT 센서를 적용하여
운동 데이터를 점수화하는 오락형 피트니스 프로토타입 이기 때문에
프로젝트의 주 목적은 정밀한 운동 처방보다 사용자의 경쟁심과 흥미를 유도하는 데 있습니다.
따라서 전체 헬스기구를 대상으로 하기보다는
랫풀다운, 레그프레스, 체스트프레스처럼 동작이 일정하고 점수화가 쉬운 2~3개의 기구를 우선 대상으로 설정했습니다.
사용자는 운동 결과를 점수로 확인하고, 랭킹을 통해 다른 사용자와 경쟁할 수 있습니다.
이를 통해 단순 반복 운동에 게임적 요소를 더해 헬스장 안에서 오락기기를 사용하는 듯한 흥미로운 운동 경험을 제공하는 것을 목표로 했습니다.

    ## 실행 가이드

Healthper는 운동 데이터를 측정하고 점수를 계산하는 IoT 피트니스 프로젝트입니다.
아래 순서대로 준비하면 프로젝트를 실행하거나 알고리즘 코드를 확인할 수 있습니다.

---

### 1. 실행 전에 필요한 것

이 프로젝트를 확인하려면 아래 프로그램과 장치가 필요합니다.

| 필요한 것        | 왜 필요한가요?                       |
| ------------ | ------------------------------ |
| Node.js      | JavaScript 코드를 실행하기 위해 필요합니다.  |
| npm          | 필요한 프로그램들을 설치하기 위해 필요합니다.      |
| Raspberry Pi | 센서와 모터를 제어하기 위한 장치입니다.         |
| ADXL345      | 운동할 때 움직임과 기울기를 측정하는 센서입니다.    |
| Step Motor   | 무게 조절 장치를 움직이기 위한 모터입니다.       |
| Git          | 코드를 GitHub에 올리고 관리하기 위해 사용합니다. |

---

### 2. 설치 방법

프로젝트 폴더에서 아래 명령어를 입력합니다.

```bash
npm install
```

이 명령어는 프로젝트 실행에 필요한 파일들을 설치하는 명령어입니다.

---

### 3. 기본 설정값

프로젝트를 실행할 때 사용할 수 있는 기본 설정값입니다.

| 이름                     | 의미             | 기본값              |
| ---------------------- | -------------- | ---------------- |
| `PORT`                 | 서버가 실행되는 번호    | `3000`           |
| `EQUIPMENT_ID`         | 운동기구 이름        | `latpulldown_01` |
| `SENSOR_ID`            | 센서 이름          | `sensor_01`      |
| `WEIGHT_STEP`          | 무게를 몇 kg씩 조절할지 | `5`              |
| `DEFAULT_POSTURE_RATE` | 자세 점수 기본값      | `90`             |

예시 설정은 아래와 같습니다.

```env
PORT=3000
EQUIPMENT_ID=latpulldown_01
SENSOR_ID=sensor_01
WEIGHT_STEP=5
DEFAULT_POSTURE_RATE=90
```

현재 저장소는 포트폴리오용으로 핵심 알고리즘과 데이터 흐름을 보여주는 목적이기 때문에, 실제 실행 환경이 없어도 코드 구조를 확인할 수 있습니다.

---

### 4. 실행 방법

서버를 실행하려면 아래 명령어를 입력합니다.

```bash
node server.js
```

서버가 실행되면 브라우저에서 아래 주소로 접속합니다.

```text
http://localhost:3000
```

실행 흐름은 다음과 같습니다.

1. 서버를 실행합니다.
2. 사용자가 웹 화면에 접속합니다.
3. 사용자가 운동 무게를 선택합니다.
4. 센서가 운동 데이터를 측정합니다.
5. 점수 알고리즘이 운동 점수를 계산합니다.
6. 결과가 저장되고 랭킹에 반영됩니다.

---

### 5. 코드 테스트 방법

현재 저장소에는 핵심 알고리즘 코드가 들어 있습니다.

| 파일명                    | 확인할 수 있는 내용                    |
| ---------------------- | ------------------------------ |
| `weightScore.js`       | 1RM 기반 운동 점수 계산                |
| `autoWeightControl.js` | 점수에 따른 무게 증가, 유지, 감소 판단        |
| `sessionMapping.js`    | 사용자, 운동기구, 센서 데이터를 하나로 연결하는 구조 |

아래 명령어로 각 코드를 실행할 수 있습니다.

```bash
node weightScore.js
node autoWeightControl.js
node sessionMapping.js
```

실행하면 터미널에 계산 결과가 출력됩니다.

### 6. 문제가 생겼을 때

| 문제                      | 왜 생기나요?                              | 해결 방법                                                |
| ----------------------- | ------------------------------------ | ---------------------------------------------------- |
| `node` 명령어가 안 됨         | Node.js가 설치되지 않았을 수 있습니다.            | Node.js를 설치하고 터미널을 다시 켭니다.                           |
| `npm install`이 안 됨      | 필요한 설정 파일이 없거나 인터넷 연결 문제가 있을 수 있습니다. | `package.json` 파일과 인터넷 연결을 확인합니다.                    |
| 센서 데이터가 사용자 기록과 연결되지 않음 | 사용자, 센서, 운동기구 정보가 연결되지 않았을 수 있습니다.   | `sessionId`, `sensorId`, `equipmentId`가 생성되는지 확인합니다. |
| 점수가 이상하게 나옴             | 무게, 반복 횟수, 자세 점수 입력값이 잘못됐을 수 있습니다.   | `weightScore.js`의 입력값을 확인합니다.                        |
| 무게 추천 결과가 이상함           | 점수 기준이나 자세 완성도 기준이 맞지 않을 수 있습니다.     | `autoWeightControl.js`의 조건문을 확인합니다.                  |
| 모터가 움직이지 않음             | 배선이나 GPIO 번호가 잘못됐을 수 있습니다.           | Raspberry Pi와 모터 드라이버 연결을 확인합니다.                     |
| 영상이 바로 안 열림             | GitHub에서 mp4 미리보기가 안 될 수 있습니다.       | 영상 파일을 다운로드해서 확인합니다.                                 |
